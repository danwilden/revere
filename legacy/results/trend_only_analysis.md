## Observations

### 1. The strategy has real OOS edge — the in-sample numbers were not pure overfitting

USD_JPY OOS Sharpe **0.816** exceeds the in-sample 0.795.
USD_CHF OOS Sharpe **0.542** declines from 0.804, but remains clearly positive.

**Conclusion**: TrendFollowStrategy + D1 gate has genuine OOS edge on H4 USD_JPY.
The in-sample/OOS gap is moderate (−0.262 CHF, +0.021 JPY), not the catastrophic
collapse typically seen from overfitted strategies.

### 2. Signal frequency is still high (47%), but trade count is reasonable

47% signal frequency sounds alarming but is structurally different from the RegimeRouter's
25% problem:

- RegimeRouter: persistent signal on every TRENDING bar → engine enters/re-enters
  frequently, many whipsaw entries per trend episode.
- TF + D1 gate: same persistent EMA signal within a trend, but the engine does NOT
  re-enter when already in a position (signal=1 while long → no action). The engine
  only acts on signal CHANGES (flat→long, long→short, etc.).

Result: 551 trades over ~8,189 evaluation bars = 6.7% trade rate (vs RegimeRouter's
~25% effective re-entry rate from its 462–469 trades in ~1,899 per fold).

The high signal frequency here is informational overhead in the signal_df, not
excessive trading activity.

### 3. Hit rate is high but payoff ratio is inverted

| | USD_JPY | USD_CHF |
|-|---------|---------|
| Hit Rate | 63.0% | 61.2% |
| Avg Win (pips) | 57.4 | 34.0 |
| Avg Loss (pips) | 85.0 | 48.3 |
| Implied payoff ratio | 0.68 | 0.70 |

High hit rate + sub-1 payoff ratio is structurally consistent with a trend-following
strategy that exits via trail stop (capturing partial trend moves) and occasionally
rides losers until the structural stop fires.

Sharpe is positive because the high hit rate (63%) overcomes the unfavorable payoff.
Kelly fraction ≈ (0.63 - 0.37/0.68) = 0.086 — modest but positive.

### 4. MaxDD is elevated vs CONFIG 4 baseline

| Config | JPY Sharpe | CHF Sharpe | JPY MaxDD | CHF MaxDD |
|--------|-----------|-----------|----------|----------|
| CONFIG 4 (SignalAggregator + struct stops + min_hold=5) | -0.216 | +1.158 | -2.8% | -1.0% |
| This run (TF alone + D1 gate + struct stops + min_hold=5 + trail) | +0.816 | +0.542 | -6.3% | -5.2% |

This run achieves higher JPY Sharpe (+0.816 vs -0.216) but at the cost of larger
drawdowns (-6.3% vs -2.8%).  The CONFIG 4 baseline has better risk-adjusted CHF
performance (+1.158 vs +0.542).

### 5. Implications for strategy development

1. **TrendFollowStrategy + D1 gate is the real signal source.** The OOS edge is
   confirmed. The strategy should be the primary building block, not one component
   in an aggregator or router.

2. **The D1 gate is load-bearing.** Without it, TF produces Sharpe -0.741 (JPY)
   and -1.011 (CHF) on the full dataset (notebook 05).  The gate is not optional.

3. **The RegimeRouter's TRENDING path must include the D1 gate.** The ranging_fix
   analysis showed RegimeRouter without D1 gate = -1.262 Sharpe (JPY).
   Re-enabling the D1 gate on TRENDING would likely recover most of that edge.

4. **CHF underperforms JPY on this strategy.** The CHF D1 regime spent 55% of time
   bearish (vs 59.6% bullish for JPY) — CHF is more USD-positive, creating more
   long blocks.  The asymmetry warrants separate parameter tuning per pair.
